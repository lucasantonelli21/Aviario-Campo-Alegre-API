## Arquitetura e Padrões de Projeto

  

*   **Estilo Arquitetural Cliente e Servidor**

  

*   **Arquitetura do Servidor: Arquitetura em Camadas**

  

*   **Definição das Camadas:**
    *   Organizada em Camada de Apresentação, Camada de Serviço, Camada de Acesso a Dados e Camada de Transferência de Dados.
    *   Camada de Apresentação (_Controllers_): Interface da API com o mundo exterior, gerenciando as requisições HTTP.
    *   Camada de Serviço: Contém a lógica de negócio, realiza validações e processa as operações de acordo com as regras da aplicação.
    *   Camada de Acesso a Dados (_Context_): Gerencia o acesso ao banco de dados usando o _Entity Framework_ ou outra ferramenta ORM.
    *   Camada de Transferência de Dados (_DTOs_): Facilita a comunicação entre a API e as camadas externas, como front-ends ou outras APIs, mantendo a integridade e a segurança dos dados.

  

*   **Padrões de Projeto:**
    *   Padrão de _Repository_:
        *   Esse padrão cria uma camada intermediária entre a aplicação e a camada de persistência (banco de dados), representando uma coleção de objetos como uma abstração das operações de dados.
        *   Têm como objetivo isolar a lógica de acesso a dados, oferecendo uma interface comum para manipular dados sem expor diretamente o _Entity Framework_ ou qualquer outra biblioteca de acesso a dados.
    *   Injeção de Dependência:
        *   Esse padrão promove a injeção de dependências externas (como repositórios, serviços, e contexto de banco de dados) nas classes que precisam delas, em vez de instanciá-las diretamente.
        *   Tem como objetivo reduzir o acoplamento entre as classes, facilitando a manutenção e os testes unitários.
    *   DTO (_Data Transfer Object_):
        *   Esse padrão garante que uma classe tenha apenas uma instância e fornece um ponto de acesso global a ela.
        *   Utilizado para serviços que precisam ser únicos durante todo o ciclo de vida da aplicação, como serviços de configuração ou cache.
